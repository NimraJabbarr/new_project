📝 QUICK SETUP (10 MINUTES)
════════════════════════════════════════════════════════════════════════════════

Step 1: Create GitHub Account (if needed)
  1. Go to https://github.com
  2. Click "Sign up"
  3. Create free account
  4. Verify email

Step 2: Create New Repository
  1. Click "+" icon (top right)
  2. Select "New repository"
  3. Name: "school-crm" (or anything)
  4. Description: "School CRM Django API"
  5. Select "Private" (keeps code secret)
  6. Click "Create repository"

Step 3: Get the Commands
  GitHub will show you setup commands
  Copy the HTTPS version:
  
  git config --global user.email "your-email@example.com"
  git config --global user.name "Your Name"
  git init
  git add .
  git commit -m "Initial commit"
  git branch -M main
  git remote add origin https://github.com/YOUR_USERNAME/school-crm.git
  git push -u origin main

Step 4: Paste Commands in Terminal
  1. Open terminal/command prompt on your computer
  2. Navigate to your project:
     cd "c:\Users\asads\OneDrive\Desktop\Day_10_School_Crm_project"
  3. Paste each command (one at a time)
  4. GitHub will ask for password (use personal access token, see below)

Step 5: Get Personal Access Token
  1. Go to GitHub Settings → Developer settings → Personal access tokens
  2. Click "Generate new token"
  3. Give it name "school-crm"
  4. Select scopes: "repo" (full control of repositories)
  5. Generate token
  6. Copy token
  7. When Git asks for password, paste token instead

🐛 TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

"Permission denied (publickey)"?
  → Use HTTPS, not SSH
  → Make sure Personal Access Token is correct

"Repository not found"?
  → Check your username is correct
  → Make sure repository is public (for Railway)

".git already exists"?
  → Your project is already a git repo
  → Skip the "git init" command

"git command not found"?
  → Download Git: https://git-scm.com/download/win
  → Install it
  → Restart terminal


💡 WHAT TO GITIGNORE
════════════════════════════════════════════════════════════════════════════════

These files should NOT go to GitHub:

db.sqlite3 (local database)
*.pyc (Python cache)
myenv/ (virtual environment)
staticfiles/ (will be rebuilt)

Check your .gitignore file - these should already be there!
